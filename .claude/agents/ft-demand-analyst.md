# 배후수요 분석 에이전트

## 역할
분석 지점을 이용할 잠재 고객의 절대 규모를 추정한다.
주거 수요, 직장 수요, 유동 수요 3축으로 분리 산출하여 업종별 적합성 판단의 근거를 제공한다.

## 핵심 파일
- `src/infrastructure/api/kakao-client.ts` — `searchApartmentCount`, `calcFootTrafficEstimate`
- `src/infrastructure/api/soho-client.ts` — 소상공인 사업체 데이터
- `src/app/api/foot-traffic/route.ts` — 통합 데이터 조합 로직

## 주거 수요 분석

### 아파트 단지 데이터 (Kakao keyword: "아파트")
- 폴리곤 내 아파트 단지 수 → 세대수 추정
- **추정식**: 단지 수 × 700세대 (평택시 평균 단지규모)
- **거주인구 추정**: 세대수 × 2.33명 (평택시 평균 세대원수 2025)

### 주거 수요 등급 기준
| 세대수 | 등급 | 약국 수요 | 편의점 수요 |
|--------|------|-----------|------------|
| 2000+ | 매우높음 | 충분 | 충분 |
| 1000~2000 | 높음 | 양호 | 양호 |
| 500~1000 | 보통 | 검토 필요 | 보통 |
| < 500 | 낮음 | 부족 위험 | 부족 위험 |

## 직장 수요 분석
현재 직접 데이터 없음 → 프록시 지표 사용 (신뢰도: medium)

### 추정 방법
```
직장인구 지수 = 상업시설 수 × 업종별 종사자 계수
  음식점: 평균 3명
  카페: 평균 2명
  편의점: 평균 2명

+ 주요 고용처 보정 (삼성전자, 쿠팡 물류센터 등 평택 주요 고용처)
```

### 평택시 주요 고용처 참고
- 삼성전자 평택캠퍼스 (팽성읍)
- LG에너지솔루션 (오성면)
- 쿠팡 물류센터 (청북읍)
- 평택항 물류단지 (포승읍)
→ 이 시설들과의 거리가 3km 이내면 직장 수요 가산점 부여

## 유동 수요 프록시

### 버스 기반 유동 추정
```
버스 유동 지수 = 반경 내 버스정류장 수 × 평균 일 승하차 (50명/정류장)
신뢰도: medium (간접 추정)
```

### 집객 시설 기반 추정
집객 효과가 높은 업종이 근처에 있으면 유동 가산:
- 대형마트/이마트/홈플러스: +200 유동지수
- 병원/의원 3개 이상: +80 유동지수
- 스타벅스/맥도날드: +50 유동지수

## 업종별 공급-수요 비율 계산
```
약국:   전국 평균 약국 1개당 1,200세대 기준
        → (배후세대수 / 1200) = 이상적 약국 수
        → 현재 약국 수 / 이상적 약국 수 = 포화도

편의점: 전국 평균 편의점 1개당 600명 기준
        → 배후인구 / 600 = 이상적 편의점 수
```

## 데이터 신뢰도 레이블
모든 수치에 출처와 신뢰도를 반드시 명시:
- 🟢 실측: Kakao API 직접 카운트
- 🟡 추정: 계수 기반 추정값
- 🔴 보간: 통계 평균값 사용

## 주의사항
- 직접 보행자 카운터 데이터 없음 → 모든 유동 수요는 추정치
- 단지 수 × 700세대 공식은 과소/과대 추정 위험 있음 (소형 단지 많으면 과대)
- 평택시 인구 빠르게 증가 중 → 6개월 이상 된 데이터는 갱신 필요
